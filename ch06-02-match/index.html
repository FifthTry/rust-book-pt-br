Operador match de Controle de Fluxo

O Rust tem um excelente operador de controle de fluxo chamado match, que nos permite comparar um valor com uma srie de padres e executar um cdigo com base no padro que casar. Padres podem ser compostos de valores literais, variveis, caracteres-curinga e vrias outras coisas. O Captulo 18 aborda todos os tipos de padres e o que eles fazem. A grande utilidade do match vem da expressividade dos padres e das anlises feitas pelo compilador, tendo certeza de que todos os casos possveis esto sendo tratados.

Imagine que expresso match funciona como uma mquina de contar moedas: as moedas passam por um canal que possui furos de vrios tamanhos, e cada moeda cai no primeiro furo em que ela couber. Da mesma forma, os valores passam por cada padro de um match, e logo no primeiro padro que o valor se encaixar, o bloco de cdigo que estiver associado a ele ser executado.

Aproveitando que acabamos de falar sobre moedas, vamos us-las como exemplo de utilizao do match! Podemos escrever uma funo que recebe uma moeda qualquer dos Estados Unidos e, assim como uma mquina, determina qual moeda ela e retorna seu valor em cents, como mostra a Listagem 6-3:

Nota do tradutor: diferentemente do que acontece na maioria dos pases, as moedas dos Estados Unidos possuem nomes: as de 1 cent so chamadas de Penny; as de 5 cents, de Nickel; as de 10 cents, de Dime; e as de 25 cents, de Quarter.

enum Moeda {
    Penny,
    Nickel,
    Dime,
    Quarter,
}

fn valor_em_cents(moeda: Moeda) -> u32 {
    match moeda {
        Moeda::Penny => 1,
        Moeda::Nickel => 5,
        Moeda::Dime => 10,
        Moeda::Quarter => 25,
    }
}

}

Listagem 6-3: Uma enum e uma expresso match em que os padres comparados so as variantes da enum.
Vamos analisar o match da funo valor_em_cents. Primeiro, usamos a palavra-chave match seguida de uma expresso, que neste caso o valor moeda. parecido a uma expresso utilizada com if, mas tem uma grande diferena: com if, a expresso precisa retornar um valor booleano. Aqui, pode ser de qualquer tipo. O tipo da varivel moeda, neste exemplo, a enum Moeda, que definimos na Listagem 6-3.
Em seguida vm os braos do match. Um brao composto por duas partes: um padro e algum cdigo. O primeiro brao deste exemplo tem um padro, que o valor Moeda::Penny, e o operador => que separa o padro do cdigo associado a ele. O cdigo, nesse caso, apenas o valor 1. Os braos so separados uns dos outros com uma vrgula.
Quando o match executado, ele compara o valor resultante com o padro de cada brao, na ordem. Se o valor casar com um determinado padro, o cdigo associado a esse padro ser executado. Se o valor no se encaixa nesse padro, a execuo passa para o prximo brao, bem parecido com a mquina de contar moedas. Podemos ter tantos braos quanto precisamos. No caso da Listagem 6-3, nosso match tem quatro braos.
O cdigo associado a cada brao uma expresso, e o seu valor resultante, no brao que combinar, o que ser retornado pela expresso match.
Tipicamente no se usa chaves se o brao do match for curto, como o caso na Listagem 6-3, em que cada brao retorna apenas um valor. Se voc quiser executar mais de uma linha de cdigo em um brao, voc pode usar chaves para delimit-las. Por exemplo, o cdigo seguinte vai escrever na tela Moeda da sorte! sempre que o mtodo for chamado com uma Moeda::Penny, mas ainda vai retornar o ltimo valor do bloco, 1:
# enum Moeda {
#    Penny,
#    Nickel,
#    Dime,
#    Quarter,
# }
#

    match moeda {
        Moeda::Penny => {
            println!("Moeda da sorte!");
            1
        },
        Moeda::Nickel => 5,
        Moeda::Dime => 10,
        Moeda::Quarter => 25,
    }
}

Padres Atrelados a Valores

Outra caracterstica til dos braos do match que eles podem ser atrelados a partes dos valores que se encaixam no padro. assim que podemos extrair valores dentro de uma variante de uma enum.

Por exemplo, vamos alterar uma das nossas variantes, inserindo dados dentro dela. De 1999 at 2008, os Estados Unidos cunhou quarters com um design diferente para cada um dos 50 estados em um dos lados da moeda. Nenhuma outra moeda tinha essa diferena no design, apenas os quarters. Podemos adicionar essa informao nossa enum alterando a variante Quarter para incluir o valor Estado, como feito na Listagem 6-4:

#[derive(Debug)] // Para podermos ver qual  o estado com mais facilidade
enum Estado {
    Alabama,
    Alaska,
    // ... etc
}

enum Moeda {
    Penny,
    Nickel,
    Dime,
    Quarter(Estado),
}

    Penny,

    Dime,
    Quarter(Estado),

Vamos imaginar que um amigo nosso est tentando colecionar todas os quarters dos 50 estados. Enquanto separamos nosso troco por tipo de moeda, vamos tambm dizer o nome do estado associado a cada quarter. Se for um dos que o nosso amigo ainda no tem, ele pode coloc-lo na sua coleo.
Na expresso match desse cdigo, vamos adicionar uma varivel chamada estado ao padro que casa com os valores da variante Moeda::Quarter. Quando uma Moeda::Quarter testada, a varivel estado vai ser atrelada ao valor do estado daquele quarter. Assim vamos poder usar o estado no cdigo do brao, desse jeito:

Quando comparamos esse valor em cada um dos braos do match, nenhum deles vai casar enquanto no chegar em Moeda::Quarter(estado). Nesse ponto, estado vai estar atrelado ao valor Estado::Alaska. Podemos, ento, usar esse valor na expresso println!, obtendo o estado contido na variante Quarter da enum Moeda.
Usando match com Option<T>

Na seo anterior, queramos obter o valor T contido em um Some quando era o caso em uma Option<T>. Tambm podemos manipular uma Option<T> usando match, assim como fizemos com a enum Moeda! Em vez de comparar moedas, vamos comparar as variantes de Option, mas a forma de trabalhar com a expresso match` continua a mesma.

Digamos que queremos escrever uma funo que recebe um Option<i32>, e se houver um valor embutido nele, soma um a esse valor. Se no houver um valor, a funo deve retornar None, e nem tentar executar nenhuma operao.

Essa funo bem fcil de implementar, graas ao match, e vai ficar conforme visto na Listagem 6-5:

fn mais_um(x: Option<i32>) -> Option<i32> {
    match x {
        None => None,
        Some(i) => Some(i + 1),
    }
}

let cinco = Some(5);
let seis = mais_um(cinco);
let nenhum = mais_um(None);

}

let seis = mais_um(cinco);
let nenhum = mais_um(None);

Voc ver muito esse padro em Rust: fazer o match de uma enum, associar uma varivel ao valor embutido, e ento executar um cdigo baseado nele. Pode parecer complicado no comeo, mas uma vez que voc se acostume, voc vai querer que tivesse isso em todas as linguagens. , sistematicamente, um favorito dos usurios.
Matches So Exaustivos
H outro aspecto do match que precisamos discutir. Considere essa verso da nossa funo mais_um:
fn mais_um(x: Option<i32>) -> Option<i32> {
    match x {
        Some(i) => Some(i + 1),
    }
}

fn mais_um(x: Option<i32>) -> Option<i32> {
    match x {
        Some(i) => Some(i + 1),
    }
}
Ns no tratamos o caso None, logo vai ocorrer um bug no nosso cdigo. Por sorte, um bug que o Rust sabe detectar. Se tentarmos compilar esse cdigo, vamos ter esse erro:

O Rust sabe que ns no cobrimos todos os casos possveis, e sabe at de qual padro nos esquecemos! Matches em Rust so exaustivos: precisamos extinguir at a ltima possibilidade pra que o nosso cdigo seja vlido. Especialmente no caso de uma Option<T>, em que o Rust no nos deixa esquecer de tratar explicitamente o caso None. Ele nos impede de assumir que temos um valor vlido quando possivelmente temos um valor nulo, e portanto, cometer o erro de um bilho de dlares que vimos mais cedo.
The _ Placeholder
O Placeholder _
O Rust tambm tem um padro que podemos usar em situaes em que no queremos listar todos os valores possveis. Por exemplo, um u8 pode ter valores vlidos de 0 a 255. Se nos importamos apenas com os valores 1, 3, 5 e 7, no queremos ser obrigados a listar o 0, 2, 4, 6, 8, 9, e assim por diante at 255. Felizmente, nem precisamos: em vez disso, podemos usar o padro especial _.
let algum_valor_u8 = 0u8;
match algum_valor_u8 {
    1 => println!("um"),
    3 => println!("trs"),
    5 => println!("cinco"),
    7 => println!("sete"),
    _ => (),
}

let algum_valor_u8 = 0u8;
match algum_valor_u8 {
    1 => println!("um"),
    3 => println!("trs"),
    5 => println!("cinco"),
    7 => println!("sete"),
    _ => (),
}
O padro _ casa com qualquer valor. Colocando ele depois dos demais braos, o _ vai casar com todos os casos possveis que no foram especificados antes dele. O () s o valor-unidade, pra que nada acontea no caso _. Como resultado, podemos dizer que no queremos fazer nada com os possveis valores que no listamos antes do placeholder _.
Contudo, a expresso match pode ser um tanto verbosa em uma situao em que queremos apenas lidar com um dos casos. Pra essa situao, o Rust oferece o if let.
