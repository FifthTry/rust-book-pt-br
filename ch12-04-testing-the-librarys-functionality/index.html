Desenvolvendo a Biblioteca de Funcionalidades com Desenvolvimento Guiado por Testes

Agora que extramos a lgica em src/lib.rs e deixamos o argumento de coleta e tratamento de erros em src/main.rs, muito mais fcil escrever testes para nosso cdigo da funcionalidade principal. Podemos chamar funes diretamente com vrios argumentos e verificar valores de retorno sem ter que chamar o nosso binrio da linha de comando. Sinta-se livre para escrever alguns testes para as funes Config::new e run por sua conta.

Nesta seo, adicionaremos a lgica de busca ao programa minigrep usando o processo Desenvolvimento Guiado por Testes (Test Driven Development (TDD)). Nessa tcnica de desenvolvimento de software, segue estas etapas:

  1. Escreva um teste que falha e execute-o, para certificar-se de que ele falha pelo motivo esperado por voc.
  2. Escreva ou modifique o cdigo apenas o suficiente para fazer passar no teste.
  3. Refatore o cdigo que voc acabou de adicionar ou alterou e certifique-se de que os testes continuam a passar.
  4. Repita a partir do passo 1!

Este processo apenas uma das muitas maneiras de escrever software, mas o TDD pode ajudar a conduzir design de cdigo tambm. Escrevendo o teste antes de escrever o cdigo que faz o teste passar, ajuda a manter uma alta cobertura de teste ao longo do processo.

Testaremos a implementao da funcionalidade que realmente far a busca da string de consulta no contedo do arquivo, e produzir uma lista de linhas que correspondem consulta. Vamos adicionar essa funcionalidade em uma funo chamada search.

Observe que precisamos de uma lifetime explcita 'a definida na assinatura do search e usada com o argumento contents e o valor de retorno. Lembre-se no Captulo 10 que os parmetros de lifetime especificam qual o lifetime do argumento conectado ao lifetime do valor de retorno. Neste caso, indicamos que o vetor retornado deve conter pedaos de string que fazem referncia a pedaos do argumento contents (em vez do argumento query).
Em outras palavras, dizemos ao Rust que os dados retornados pela funo search ir viver enquanto os dados passarem para a funo search no argumento de contents. Isso importante! Os dados referenciados por um pedao precisa ser vlido para que a referncia seja vlida; se o compilador assume que estamos fazendo pedaos de string de query em vez de contents, ele far sua verificao de segurana incorretamente.

Escrevendo Cdigo para Passar no Teste

Atualmente, nosso teste est falhando porque sempre devolvemos um vetor vazio. Para consertar isso preciso implementar search, nosso programa precisa seguir essas etapas:

  • Iterar atravs de cada linha do contedo.
  • Verificar se a linha contm nossa string de consulta.
  • Se a tiver, adicione-a lista de valores que estamos retornando.
  • Se no, no faa nada.
  • Retorna a lista de resultados que correspondem.

Vamos trabalhar em cada passo, comeando por iterar atravs de linhas.

Armazenamento de Linhas Correspondentes

Ns tambm precisamos de uma maneira de armazenar as linhas que contm nossa string de consulta. Por isso, podemos fazer um vetor mutvel antes do loop for e chamar o mtodo push para armazenar uma line no vetor. Aps o loop for, devolvemos o vetor, como mostrado na Listagem 12-19:

Arquivo: src/lib.rs

pub fn search<'a>(query: &str, contents: &'a str) -> Vec<&'a str> {
    let mut results = Vec::new();

    for line in contents.lines() {
        if line.contains(query) {
            results.push(line);
        }
    }

    results
}

pub fn search<'a>(query: &str, contents: &'a str) -> Vec<&'a str> {
    let mut results = Vec::new();

    for line in contents.lines() {
        if line.contains(query) {
            results.push(line);
        }
    }

    results
}

Usando a Funo search na Funo run

Agora que a funo search est funcionando e testada, precisamos chamar search da nossa funo run. Precisamos passar o valor config.query e o contents que run l do arquivo para a funo search. Ento, run ir imprimir cada linha retornada de search:

Arquivo: src/lib.rs

pub fn run(config: Config) -> Result<(), Box<Error>> {
    let mut f = File::open(config.filename)?;

    let mut contents = String::new();
    f.read_to_string(&mut contents)?;

    for line in search(&config.query, &contents) {
        println!("{}", line);
    }

    Ok(())
}
